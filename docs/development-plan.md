# 2. 首次启动欢迎页

## 2.1 页面目标

引导第一次使用软件的用户添加本地项目。

## 2.2 页面原型

```
┌─────────────────────────────────────────────────────────┐
│                  本地 Git 项目管理器                     │
│                                                         │
│        集中管理电脑中的 Git 项目，并快速使用 IDE 打开    │
│                                                         │
│  你可以扫描指定目录，也可以手动添加单个 Git 项目。       │
│                                                         │
│              [ 选择扫描目录 ]                           │
│                                                         │
│              [ 手动添加项目 ]                           │
│                                                         │
│  所有项目数据只保存在当前电脑中，不会上传。              │
└─────────────────────────────────────────────────────────┘
```

## 2.3 交互规则

点击“选择扫描目录”：

- 打开 Windows 文件夹选择器。
    
- 支持连续添加多个目录。
    
- 添加完成后进入扫描目录设置页。
    

点击“手动添加项目”：

- 打开文件夹选择器。
    
- 检查选中目录是否为 Git 仓库。
    
- 检查通过后直接进入导入确认流程。

# 3. 扫描目录设置页

## 3.1 页面目标

让用户确认需要扫描的根目录。

## 3.2 页面原型

```
┌─────────────────────────────────────────────────────────┐
│ ← 返回                选择扫描目录                       │
├─────────────────────────────────────────────────────────┤
│                                                         │
│ 将在以下目录中查找 Git 项目：                            │
│                                                         │
│  D:\Projects                              [移除]         │
│  E:\Workspace                             [移除]         │
│                                                         │
│  [ + 添加目录 ]                                          │
│                                                         │
│  ☑ 使用默认忽略规则                                      │
│                                                         │
│              [ 开始扫描 ]                               │
└─────────────────────────────────────────────────────────┘
```

## 3.3 校验规则

点击“开始扫描”前：

- 至少存在一个扫描目录。
    
- 扫描目录不能完全重复。
    
- 路径不存在时显示错误。
    
- 路径无权限时显示警告。
    
- 一个目录包含另一个目录时允许保存，但提示可能重复扫描。

# 9. Git 冲突状态页面

项目发生冲突后，详情页顶部显示明显状态：

```
┌────────────────────────────────────────────────┐
│ ⚠ 当前项目存在 Git 冲突                        │
│                                                │
│ Pull 操作未完成，共发现 4 个冲突文件。          │
│ 请使用 IDE 或终端解决冲突。                    │
│                                                │
│ [使用 IntelliJ IDEA 打开] [在终端中打开]       │
│ [刷新状态]                                     │
└────────────────────────────────────────────────┘
```

存在冲突期间：

- 禁用再次 Pull。
    
- 允许查看分支。
    
- 不允许直接切换分支。
    
- 不提供冲突解决按钮。
    
- 用户处理后点击“刷新状态”。

# 14. 软件模块划分

## 14.1 界面模块

负责：

- 页面显示
    
- 用户输入
    
- 状态提示
    
- 弹窗
    
- 主题
    
- 页面导航
    

不直接执行 Git 命令或操作数据库。

## 14.2 项目管理模块

负责：

- 添加项目
    
- 移除项目
    
- 修改显示名称
    
- 更新项目路径
    
- 最近打开排序
    
- 项目搜索
    
- 项目去重
    

## 14.3 扫描模块

负责：

- 遍历扫描目录
    
- 应用忽略规则
    
- 识别 Git 仓库
    
- 识别嵌套仓库
    
- 处理无权限目录
    
- 输出扫描结果
    

## 14.4 Git 服务模块

负责：

- 检测 Git
    
- 获取当前分支
    
- 获取未提交修改数量
    
- 获取最近提交
    
- 获取远程仓库
    
- Fetch
    
- 计算领先和落后数量
    
- 获取分支列表
    
- 切换分支
    
- Pull
    
- 检测冲突
    
- 格式化 Git 错误
    

所有 Git 命令必须通过该模块统一执行。

## 14.5 IDE 管理模块

负责：

- 自动检测 IDE
    
- 管理手动添加 IDE
    
- 检查 IDE 可用性
    
- 调用 IDE 打开项目
    
- 读取 IDE 图标
    
- 管理项目默认 IDE
    

## 14.6 本地存储模块

负责保存：

- 项目记录
    
- 扫描目录
    
- Git 状态缓存
    
- IDE 配置
    
- 用户设置
    
- 最近一次 Git 操作
    

## 14.7 后台任务模块

负责：

- 项目状态刷新队列
    
- 并发控制
    
- 任务取消
    
- 命令超时
    
- 当前详情项目优先刷新
    
- 扫描进度通知
    

## 14.8 系统集成模块

负责：

- 系统托盘
    
- Windows 文件夹选择器
    
- 文件资源管理器
    
- PowerShell 或 Windows Terminal
    
- Windows 主题检测
    
- 安装目录和数据目录管理
    

## 14.9 更新模块

负责：

- 检查版本
    
- 获取更新说明
    
- 下载更新包
    
- 用户确认
    
- 安装新版
    
- 保留配置数据

# 15. Git 命令能力清单

Git 服务模块至少需要封装以下能力：

```
检测 Git 是否存在
获取 Git 版本
判断目录是否为 Git 仓库
获取仓库根目录
获取当前分支
获取工作区修改文件数量
获取最近提交信息
获取远程仓库地址
获取本地分支
获取远程分支
获取上游分支
获取 Ahead / Behind
执行 Fetch
切换本地分支
创建远程跟踪分支
执行 Pull --rebase
执行 Pull --no-rebase
执行普通 Pull
检测 Merge 或 Rebase 冲突
统计冲突文件数量
```

明确不提供：

```
Commit
Push
Force Push
Reset
Clean
Checkout 文件
删除分支
创建普通本地分支
Stash
Rebase Continue
Rebase Abort
Merge Continue
Merge Abort
```

# 16. 开发阶段拆分

## 阶段一：基础工程和本地数据

目标：建立可运行的桌面应用框架。

任务：

- 创建桌面应用工程
    
- 实现主窗口
    
- 实现页面导航
    
- 建立本地数据库
    
- 实现全局设置保存
    
- 实现浅色和深色主题
    
- 建立统一错误处理机制
    

完成标准：

- 软件可以启动和退出。
    
- 设置修改后重新打开仍然存在。
    
- 页面可以正常切换。
    

## 阶段二：项目扫描和项目列表

目标：用户可以导入并查看本地 Git 项目。

任务：

- 添加扫描目录
    
- 实现递归目录扫描
    
- 实现忽略规则
    
- 识别 Git 仓库
    
- 识别嵌套仓库
    
- 扫描结果勾选和导入
    
- 手动添加项目
    
- 项目去重
    
- 项目卡片视图
    
- 项目表格视图
    
- 项目搜索
    
- 最近打开排序
    
- 路径失效检查
    

完成标准：

- 可以从指定目录导入多个 Git 项目。
    
- 重复扫描不会产生重复数据。
    
- 可以搜索和进入项目详情。
    

## 阶段三：IDE 检测和项目打开

目标：用户可以选择 IDE 打开项目。

任务：

- 检测 JetBrains Toolbox
    
- 检测 JetBrains 独立安装版本
    
- 检测 VS Code
    
- 手动添加 IDE
    
- IDE 可用性检查
    
- 首次打开 IDE 选择
    
- 项目默认 IDE
    
- 临时使用其他 IDE
    
- 在资源管理器中打开
    
- 在终端中打开
    
- 最近打开时间更新
    

完成标准：

- 可以成功使用 IDEA 和 VS Code 打开项目。
    
- 每个项目能够记住不同的默认 IDE。
    
- IDE 路径失效时有明确提示。
    

## 阶段四：Git 本地状态

目标：查看项目基础 Git 状态。

任务：

- 检测 Git
    
- 获取当前分支
    
- 获取未提交修改数量
    
- 获取最近提交
    
- 获取远程仓库地址
    
- Git 状态缓存
    
- 启动时后台刷新
    
- 当前详情项目优先刷新
    
- 刷新失败处理
    
- Git 命令超时机制
    

完成标准：

- 首页可以快速展示缓存。
    
- 后台刷新不会卡住界面。
    
- 单个项目失败不会影响其他项目。
    

## 阶段五：Fetch、分支和 Pull

目标：完成 MVP 的 Git 操作能力。

任务：

- 手动 Fetch
    
- 领先和落后数量计算
    
- 本地分支列表
    
- 远程分支列表
    
- 分支搜索
    
- 本地分支切换
    
- 远程跟踪分支创建
    
- 未提交修改风险提示
    
- Pull 策略配置
    
- Pull 确认弹窗
    
- Pull 执行
    
- 冲突识别
    
- 冲突文件数量
    
- 最近一次 Git 操作记录
    

完成标准：

- 可以主动检查远程更新。
    
- 可以安全切换分支。
    
- 可以确认后执行 Pull。
    
- 冲突后能够正确提示，但不处理冲突。
    

## 阶段六：系统集成和发布

目标：达到可安装和发布状态。

任务：

- 系统托盘
    
- 关闭窗口行为
    
- 安装版打包
    
- 免安装版打包
    
- 软件更新检查
    
- 更新确认流程
    
- 错误日志
    
- 软件图标
    
- 安装和卸载测试
    
- 数据升级兼容测试
    

完成标准：

- 安装版和免安装版均可正常使用。
    
- 升级软件不会丢失项目配置。
    
- 错误日志能够定位常见失败。

# 17. 推荐开发顺序

建议不要一开始同时开发所有能力。

推荐顺序：

```
项目扫描
  ↓
项目列表和搜索
  ↓
IDE 检测与打开
  ↓
Git 本地状态
  ↓
Fetch 和分支切换
  ↓
Pull 与冲突识别
  ↓
托盘、更新和打包
```

第一个可用版本应优先实现：

```
扫描项目 → 搜索项目 → 使用 IDE 打开
```

完成这条核心链路后，再增加 Git 状态和 Pull。

# 18. 测试项目准备

开发期间建议准备以下测试仓库：

1. 正常 Git 仓库
    
2. 没有远程仓库的 Git 仓库
    
3. 存在未提交文件的仓库
    
4. 本地领先远程的仓库
    
5. 本地落后远程的仓库
    
6. 没有上游分支的仓库
    
7. 包含嵌套 Git 仓库的目录
    
8. 存在 Rebase 冲突的仓库
    
9. 路径失效的项目记录
    
10. 无访问权限的扫描目录
    
11. 中文目录名称的仓库
    
12. 路径中包含空格的仓库
    
13. 超长路径仓库
    
14. 位于移动磁盘中的仓库

# 19. MVP 完成定义

当以下流程全部稳定运行时，可以认为 MVP 完成：

1. 用户安装或解压软件。
    
2. 用户添加扫描目录。
    
3. 软件正确发现 Git 项目。
    
4. 用户确认并导入项目。
    
5. 用户可以搜索项目。
    
6. 用户可以进入项目详情。
    
7. 用户可以选择 IDE 打开项目。
    
8. 软件能够记住项目默认 IDE。
    
9. 软件可以展示基础 Git 状态。
    
10. 用户可以主动 Fetch。
    
11. 用户可以切换本地或远程分支。
    
12. 用户可以确认后 Pull。
    
13. Pull 冲突后软件可以正确标记项目。
    
14. 项目路径失效不会导致数据丢失。
    
15. 安装版与免安装版均可正常运行。
    
16. 软件不会上传项目和 Git 数据。
